# 시험 문제 생성 규칙 (RULE)

이 문서는 **시험 문제 파일을 어떤 형식으로 만들어야 하는지** 정의합니다.
AI 또는 사람이 이 문서를 그대로 따르면, 만든 파일을 사이트에 넣었을 때 곧바로 시험으로 변환됩니다.

지원 형식은 두 가지입니다.

- **TXT 형식** — 사람이 손으로 쓰기 쉬움 (`.txt`)
- **JS 형식** — 구조가 명확하고 보기별 해설까지 표현하기 좋음 (`.js`)

문제 유형은 **객관식**과 **주관식** 두 가지입니다.

---

## 1. TXT 형식

### 1-1. 권장(인라인) 형식

각 문제는 **빈 줄**로 구분합니다. 정답과 해설을 문제 바로 아래에 적습니다.

#### 객관식

```
Q1. 질문 내용을 여기에 적습니다.
① 첫 번째 보기
② 두 번째 보기
③ 세 번째 보기
④ 네 번째 보기
정답: ③
해설: 왜 ③번이 정답인지 설명합니다.
보기해설:
① 이 보기가 틀린 이유
② 이 보기가 틀린 이유
③ 이 보기가 정답인 이유
④ 이 보기가 틀린 이유
```

- 문제 번호는 `Q1.` `1.` `1)` 모두 인식합니다.
- 보기 기호는 `①②③④…` 또는 `1) 2) 3)` 형식을 사용합니다.
- `정답:` 에는 기호(`③`) 또는 숫자(`3`)를 적습니다.
- `해설:` 은 선택 사항이지만 적는 것을 권장합니다.
- `보기해설:` 은 **선택 사항**입니다. 적으면 각 보기 아래에 개별 설명이 표시됩니다.

#### 주관식

```
[주관식] 질문 내용을 여기에 적습니다.
모범답안: 모범 답안 내용을 적습니다. 여러 줄도 가능합니다.
해설: 채점 시 참고할 핵심 키워드나 보충 설명 (선택).
```

- 주관식은 맨 앞에 `[주관식]` 을 붙이거나, `모범답안:` 줄이 있으면 자동으로 주관식으로 인식됩니다.
- 보기(①②③④)가 없는 문제도 자동으로 주관식 처리됩니다.

### 1-2. 분리(레거시) 형식 — 기존 `문제.txt` 호환

문제를 모두 먼저 쓰고, 맨 아래에 `정답 및 해설` 구분선을 두는 형식도 지원합니다.

```
Q1. 질문...
① 보기  ② 보기  ③ 보기  ④ 보기

Q2. 질문...
...

=========================================================
정답 및 해설
=========================================================

Q1. 정답: ③
해설: 설명...
암기 팁: ...

Q2. 정답: ①
해설: 설명...
```

- `정답 및 해설` 이라는 문구가 들어간 줄을 기준으로 문제부와 정답부를 나눕니다.
- 정답부의 `Q1.`, `Q2.` 번호 순서대로 위 문제와 1:1로 연결됩니다.
- `암기 팁:` 은 해설에 함께 표시됩니다.

> ⚠️ 레거시 형식은 객관식만 표현할 수 있습니다. 주관식이나 보기별 해설이 필요하면 **인라인 형식이나 JS 형식**을 사용하세요.

## 2. JS 형식 (권장)

전역 배열 `QUIZ_QUESTIONS` 를 정의합니다. 가장 안전한 방법은 `window.QUIZ_QUESTIONS` 입니다.

```js
window.QUIZ_QUESTIONS = [
  // ── 객관식 ──
  {
    type: "multiple",
    question: "질문 내용",
    choices: ["보기1", "보기2", "보기3", "보기4"],
    answer: 3,                    // 정답 보기 번호 (1부터 시작: 1=①, 2=②, 3=③, 4=④)
    explanation: "전체 해설",       // 선택
    choiceExplanations: [          // 선택. 각 보기 아래에 표시
      "보기1이 틀린 이유",
      "보기2가 틀린 이유",
      "보기3이 정답인 이유",
      "보기4가 틀린 이유"
    ]
  },

  // ── 주관식 ──
  {
    type: "short",
    question: "질문 내용",
    modelAnswer: "모범 답안 (여러 줄 가능)",
    explanation: "채점 참고용 핵심 키워드 (선택)"
  }
];
```

### 필드 설명

| 필드 | 유형 | 필수 | 설명 |
|------|------|:----:|------|
| `type` | `"multiple"` / `"short"` | 권장 | 생략 시 `choices`가 있으면 객관식, 없으면 주관식 |
| `question` | 문자열 | ✅ | 문제 본문 |
| `choices` | 문자열 배열 | 객관식 ✅ | 보기 목록 |
| `answer` | 숫자 | 객관식 ✅ | 정답 보기 번호. **1부터 시작** (`1`=첫 번째 보기 ①) |
| `explanation` | 문자열 | 선택 | 전체 해설 |
| `choiceExplanations` | 문자열 배열 | 선택 | 보기 순서대로 1:1, 각 보기 아래 표시 |
| `modelAnswer` | 문자열 | 주관식 ✅ | 모범 답안 |

---

## 3. 동작 방식 요약

- **한 페이지에 한 문제씩** 표시됩니다.
- **객관식**: 보기를 선택하면 즉시 정답/오답이 색으로 표시되고, 각 보기 아래에 (있다면) 해설이 나옵니다.
- **주관식**: 답을 작성해 제출하면 모범 답안이 나오고, 사용자가 직접 `⭕ 맞음 / ❌ 틀림` 으로 채점합니다.
- 모든 문제를 풀면 **`정답 수 / 전체 문항 수` 와 정답률(%)** 이 표시됩니다.
  - 객관식은 선택한 답으로, 주관식은 자가 채점 결과로 집계됩니다.

## 4. AI에게 출제를 맡길 때 (프롬프트 예시)

> 아래 RULE.md 형식에 맞춰 `OOO` 과목 예상 문제 30개를 만들어줘.

## 5. 완성 예시 (그대로 복사해서 시작하기)

아래 두 예시는 **실제 내용이 채워진 완전한 파일 한 개**입니다.
같은 4문항(객관식 2 + 주관식 2)을 TXT와 JS 두 형식으로 똑같이 표현했습니다.
새 문제를 만들 때는 아래 예시를 통째로 복사한 뒤 내용만 바꾸세요.

### 5-1. TXT 완성 예시 (`예시.txt`)

```
Q1. 다음 중 운영체제의 핵심 기능이 아닌 것은?
① 프로세스 관리
② 메모리 관리
③ 소스 코드 컴파일
④ 파일 시스템 관리
정답: ③
해설: 컴파일은 컴파일러(개발 도구)의 역할이며, 운영체제의 핵심 기능이 아닙니다.
보기해설:
① 프로세스 생성·스케줄링은 운영체제의 핵심 기능입니다.
② 메모리 할당·회수는 운영체제가 담당합니다.
③ 소스 코드 컴파일은 컴파일러가 하는 일입니다.
④ 파일 저장·접근 관리는 운영체제의 핵심 기능입니다.

Q2. 교착 상태(Deadlock) 발생의 4가지 필요조건이 아닌 것은?
① 상호 배제
② 점유와 대기
③ 선점 가능
④ 환형 대기
정답: 3
해설: 교착 상태의 조건은 비선점(점유한 자원을 강제로 빼앗을 수 없음)입니다. "선점 가능"은 오히려 교착 상태를 깨는 조건입니다.

Q3. [주관식] 프로세스와 스레드의 차이를 자원 공유 관점에서 한 문장으로 설명하세요.
모범답안: 프로세스는 독립된 메모리 공간을 가지지만, 스레드는 같은 프로세스 안에서 코드·데이터·힙 영역을 공유한다.
해설: '독립된 메모리'와 '메모리 공유'라는 두 키워드가 들어가면 정답으로 인정합니다.

Q4. [주관식] 캐시 메모리를 사용하는 이유를 '지역성' 개념과 함께 설명하세요.
모범답안: CPU와 주기억장치의 속도 차이를 줄이기 위해, 최근 참조한 데이터가 다시 참조될 가능성이 높다는 지역성(시간·공간 지역성)을 이용해 자주 쓰는 데이터를 빠른 캐시에 두기 때문이다.
해설: 핵심 키워드 - 속도 차이, 시간 지역성, 공간 지역성.
```

> 포인트
> - 문제끼리는 **빈 줄**로 구분합니다.
> - 객관식 `정답:` 은 기호(`③`)와 숫자(`3`) 모두 됩니다(Q1·Q2 비교).
> - `보기해설:` 과 보기별 해설은 **선택**입니다(Q1엔 있고 Q2엔 없음).
> - 주관식은 `[주관식]` 라벨 또는 `모범답안:` 줄로 인식됩니다.

### 5-2. JS 완성 예시 (`예시.js`)

위 TXT와 **완전히 동일한 4문항**을 JS로 표현한 것입니다.

```js
window.QUIZ_QUESTIONS = [
  // ── Q1. 객관식 (보기별 해설 포함) ──
  {
    type: "multiple",
    question: "다음 중 운영체제의 핵심 기능이 아닌 것은?",
    choices: ["프로세스 관리", "메모리 관리", "소스 코드 컴파일", "파일 시스템 관리"],
    answer: 3, // 1부터 시작 → 3 = ③ "소스 코드 컴파일"
    explanation: "컴파일은 컴파일러(개발 도구)의 역할이며, 운영체제의 핵심 기능이 아닙니다.",
    choiceExplanations: [
      "프로세스 생성·스케줄링은 운영체제의 핵심 기능입니다.",
      "메모리 할당·회수는 운영체제가 담당합니다.",
      "소스 코드 컴파일은 컴파일러가 하는 일입니다.",
      "파일 저장·접근 관리는 운영체제의 핵심 기능입니다."
    ]
  },

  // ── Q2. 객관식 (보기별 해설 없이 전체 해설만) ──
  {
    type: "multiple",
    question: "교착 상태(Deadlock) 발생의 4가지 필요조건이 아닌 것은?",
    choices: ["상호 배제", "점유와 대기", "선점 가능", "환형 대기"],
    answer: 3, // 3 = ③ "선점 가능"
    explanation:
      "교착 상태의 조건은 비선점(점유한 자원을 강제로 빼앗을 수 없음)입니다. '선점 가능'은 오히려 교착 상태를 깨는 조건입니다."
  },

  // ── Q3. 주관식 ──
  {
    type: "short",
    question: "프로세스와 스레드의 차이를 자원 공유 관점에서 한 문장으로 설명하세요.",
    modelAnswer:
      "프로세스는 독립된 메모리 공간을 가지지만, 스레드는 같은 프로세스 안에서 코드·데이터·힙 영역을 공유한다.",
    explanation: "'독립된 메모리'와 '메모리 공유'라는 두 키워드가 들어가면 정답으로 인정합니다."
  },

  // ── Q4. 주관식 (여러 줄 모범답안) ──
  {
    type: "short",
    question: "캐시 메모리를 사용하는 이유를 '지역성' 개념과 함께 설명하세요.",
    modelAnswer:
      "CPU와 주기억장치의 속도 차이를 줄이기 위해, 최근 참조한 데이터가 다시 참조될 가능성이 높다는 " +
      "지역성(시간·공간 지역성)을 이용해 자주 쓰는 데이터를 빠른 캐시에 두기 때문이다.",
    explanation: "핵심 키워드 - 속도 차이, 시간 지역성, 공간 지역성."
  }
];
```

> 포인트
> - 반드시 **`window.QUIZ_QUESTIONS = [ ... ];`** 배열로 감쌉니다.
> - `answer` 는 **1부터** 셉니다(`3` = 세 번째 보기 ③). TXT의 `정답: ③` 과 같은 값입니다.
> - `choiceExplanations` 는 `choices` 와 **같은 순서·같은 개수**로 1:1 대응시킵니다(선택).
> - 주관식은 `choices` 없이 `modelAnswer` 를 넣으면 됩니다(`type: "short"` 권장).
> - 모범답안이 길면 위 Q4처럼 문자열을 `+` 로 이어 붙여도 됩니다.
